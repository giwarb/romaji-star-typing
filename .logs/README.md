# Work Logs

Use this directory for long-running Codex work logs.

Each log should include:

- Date and task name.
- Baseline score or failing signal.
- One focused change per iteration.
- Eval result after each change.
- Visual artifact inspected, when relevant.
- Current best score and next hypothesis.

`npm run eval:game` writes `.logs/latest-eval.md` as a rolling machine-generated summary.
